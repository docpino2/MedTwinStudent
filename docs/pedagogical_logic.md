# Pedagogical Logic for MedTwin Student

MedTwin Student models medical learning as a dynamic system: knowledge, skills, attention, uncertainty tolerance, feedback, context, and prior experience interact over time. The digital twin does not treat competence as a fixed score; it represents evolving readiness for clinical action under increasing complexity.

## Conceptual Foundations

- Medicine: biomedical mechanisms, illness scripts, differential diagnosis, risk stratification, patient safety, management, and professional communication.
- Clinical reasoning: problem representation, hypothesis generation, discriminating features, diagnostic justification, management prioritization, and reflection.
- Cognitive science: schema formation, retrieval practice, calibration, cognitive load, metacognition, spaced repetition, transfer, and bias mitigation.
- Complexity thinking: nonlinear learning trajectories, contextual performance, uncertainty, adaptive expertise, feedback loops, and emergence of competence across repeated cases.

## Scoring Rubric

The same 0 to 4 rubric applies to competencies and subcompetencies across all cycles. The interpretation becomes stricter as the learner advances.

| Score | Mastery level | Meaning | Typical evidence |
| --- | --- | --- | --- |
| 0 | Not observed | No reliable evidence or harmful misconception. | Missing response, unsafe answer, major conceptual error. |
| 1 | Novice | Fragmented recall with limited transfer to a case. | Names facts but does not connect them to mechanism or decision. |
| 2 | Developing | Partially correct reasoning with gaps, omissions, or poor prioritization. | Identifies some relevant features but misses key discriminators or urgency. |
| 3 | Competent | Coherent, safe, context-appropriate performance with minor gaps. | Builds a reasonable differential, justifies actions, uses feedback. |
| 4 | Adaptive | Flexible expert-like performance under uncertainty and complexity. | Transfers across contexts, calibrates confidence, anticipates failure points. |

## Cycle 1: Basic Sciences

### Core Competencies

- Biomedical mechanism building: anatomy, physiology, pathology, pharmacology, microbiology, genetics, and biochemistry as causal systems.
- Conceptual integration: linking molecules, cells, organs, and patient-level phenomena.
- Scientific reasoning: interpreting evidence, mechanisms, uncertainty, and model limitations.
- Early clinical relevance: connecting mechanisms to symptoms, signs, and risk.
- Metacognition: recognizing uncertainty, confidence errors, and knowledge boundaries.

### Observable Behaviors

- Explains a symptom using a plausible mechanism instead of memorized labels.
- Maps prerequisite concepts and identifies missing links.
- Distinguishes normal physiology from pathophysiologic deviation.
- Uses causal language: trigger, mediator, compensatory response, consequence.
- States confidence and asks for the concept needed to proceed.

### Assessment Methods

- Concept maps and prerequisite graph completion.
- Short mechanistic explanations.
- Retrieval practice with spaced repetition.
- Mechanism-to-symptom matching.
- Low-fidelity cases focused on causal reasoning.
- Confidence calibration checks.

### Common Failure Modes

- Memorization without causal integration.
- Isolated organ-system knowledge.
- Confusing association with causation.
- Overconfidence after recognition-based quizzes.
- Excess cognitive load when multiple mechanisms interact.

### AI-Supported Interventions

- Socratic mechanism tutor: asks why and how before what diagnosis.
- Adaptive concept-map repair.
- Retrieval scheduling based on forgetting curves.
- Analogy generation across organ systems.
- Misconception detection and targeted micro-lessons.
- Confidence calibration prompts.

### Cycle-Specific Rubric

| Score | Interpretation |
| --- | --- |
| 0 | Cannot recall or explains with unsafe misconception. |
| 1 | Recalls isolated facts without mechanistic connection. |
| 2 | Builds partial mechanism but misses key causal steps. |
| 3 | Explains mechanism coherently and links to basic clinical presentation. |
| 4 | Transfers mechanism across contexts and predicts downstream consequences. |

## Cycle 2: Clinical Sciences

### Core Competencies

- Clinical data gathering: focused history, physical examination, and interpretation of initial tests.
- Problem representation: concise synthesis using semantic qualifiers.
- Differential diagnosis: generating, ranking, and revising diagnostic hypotheses.
- Clinical reasoning: identifying discriminating features and avoiding premature closure.
- Evidence-informed decision-making: choosing tests and interpreting results probabilistically.
- Communication: presenting cases, explaining uncertainty, and receiving feedback.

### Observable Behaviors

- Converts raw case data into a meaningful problem representation.
- Prioritizes dangerous diagnoses before common benign diagnoses.
- Justifies the top differential using supporting and opposing evidence.
- Selects tests based on pretest probability and consequences.
- Updates reasoning after new data.
- Names cognitive bias risks in their own reasoning.

### Assessment Methods

- Script concordance tests.
- Key-feature questions.
- Objective structured clinical examinations.
- Case presentations with rubrics.
- Think-aloud reasoning traces.
- AI-generated case variants with feedback.
- Mini-CEX style workplace observations.

### Common Failure Modes

- Premature closure on the first plausible diagnosis.
- Anchoring on a salient feature.
- Broad but unprioritized differential diagnosis.
- Test ordering without clinical question.
- Poor calibration of confidence.
- Difficulty transferring textbook knowledge to messy cases.

### AI-Supported Interventions

- Differential diagnosis coach.
- Illness script comparison.
- Cognitive forcing strategies.
- Case contrast sets: similar presentations, different diagnoses.
- Feedback on problem representation quality.
- Socratic prompts for missing discriminating features.

### Cycle-Specific Rubric

| Score | Interpretation |
| --- | --- |
| 0 | Unsafe or absent clinical reasoning. |
| 1 | Lists diagnoses or facts without prioritization. |
| 2 | Partially prioritizes but misses key danger signs or discriminators. |
| 3 | Produces safe, coherent differential and next diagnostic steps. |
| 4 | Adapts reasoning as data evolves and explicitly manages uncertainty and bias. |

## Cycle 3: Internship

### Core Competencies

- Situated clinical performance: acting safely in real or simulated care contexts.
- Management prioritization: stabilization, disposition, escalation, and follow-up.
- Interprofessional communication: concise handoffs, closed-loop communication, and team coordination.
- Patient safety: recognizing deterioration, uncertainty, and system constraints.
- Adaptive expertise: balancing protocols, patient variability, and resource constraints.
- Reflective practice: learning from outcomes, feedback, and near misses.

### Observable Behaviors

- Identifies unstable patients and escalates early.
- Initiates safe first management steps while continuing diagnostic reasoning.
- Communicates assessment and plan clearly to supervisors and teams.
- Anticipates complications and contingency plans.
- Balances guideline-based care with patient-specific context.
- Reflects after action and updates future strategy.

### Assessment Methods

- Entrustable Professional Activity observations.
- Simulation scenarios with time pressure.
- Chart review and management-plan audit.
- Supervisor feedback and multisource feedback.
- Case logs linked to competency evidence.
- Post-case reflection and error analysis.
- AI-assisted longitudinal learning trajectory review.

### Common Failure Modes

- Delayed escalation.
- Task fixation under pressure.
- Overreliance on early negative tests.
- Management plans disconnected from differential diagnosis.
- Communication gaps during handoff.
- Failure to adapt when the clinical context changes.

### AI-Supported Interventions

- High-fidelity branching simulations.
- Escalation and disposition coaching.
- Handoff rehearsal with structured feedback.
- Cognitive load-aware task sequencing.
- Near-miss debrief generator.
- Longitudinal dashboard for supervisors.

### Cycle-Specific Rubric

| Score | Interpretation |
| --- | --- |
| 0 | Unsafe action or failure to recognize risk. |
| 1 | Recognizes fragments of the situation but cannot act safely without direct prompting. |
| 2 | Performs some safe actions but misses prioritization, escalation, or communication elements. |
| 3 | Manages common cases safely with appropriate supervision and clear reasoning. |
| 4 | Performs adaptively in complex contexts, anticipates deterioration, and coordinates care. |

## Competency Model

The MVP competency model should represent each competency as a graph-connected learning object with observable evidence. A competency can contain subcompetencies, each of which can be scored independently and linked to remediation strategies.

### Recommended Top-Level Competency Families

- Biomedical mechanisms
- Data gathering and interpretation
- Problem representation
- Differential diagnosis
- Diagnostic testing and probabilistic reasoning
- Initial management and patient safety
- Communication and collaboration
- Metacognition and cognitive bias mitigation
- Adaptive expertise under complexity

### Evidence Sources

- Quiz item
- Concept map
- Case response
- Think-aloud transcript
- OSCE station
- Simulation event log
- Supervisor observation
- Chart audit
- Reflection note
- AI tutor dialogue

### Remediation Strategy Types

- Mechanism micro-lesson
- Retrieval practice set
- Concept map repair
- Illness script comparison
- Socratic tutoring
- Cognitive forcing checklist
- Case contrast set
- Simulation retry
- Handoff rehearsal
- Supervisor review
